# Criando um Aplicativo

Para acessar a API Pública, é necessário criar um aplicativo na conta do desenvolvedor principal do projeto. Para cadastrar seu aplicativo e gerar uma credencial de API, visite [https://console.eduzz.com](https://console.eduzz.com).

No menu principal, selecione a opção **Meus Aplicativos** e clique no botão **Novo Aplicativo**, então, preencha todos os campos obrigatórios e selecione os escopos que seu aplicativo irá utilizar.

> #### Para determinar os escopos necessários, consulte a documentação de cada endpoint que seu aplicativo irá utilizar.

![Consultar escopo](/docs/images/credentials/scopes.png)

Então, preencha os dados do seu aplicativo no [console](https://console.eduzz.com).

![Novo App](/docs/images/credentials/new-app.png)

Ao clicar no botão **Criar Aplicativo** as credenciais do seu aplicativo serão exibidas:

![Create APP](/docs/images/credentials/credentials.png)

Copie e salve as credenciais. Caso perca a secret, será necessário gerar uma nova.

```js
clientId: 22edfacb-9abd-4dfd-a9be-31a18266aeef
clientSecret: ebf0cc8f59be0570952782d95dacb486188a9e7b2c9dea0f38a0b9aa97a8cfeb
```

Essas chaves são muito importantes. O clientId é o identificador do seu aplicativo na Eduzz, enquanto o clientSecret é o segredo do seu aplicativo, utilizado para verificar informações e também para [trocar um código de autenticação por uma chave de acesso](/docs/api/user-token).

## Status do aplicativo

O status do aplicativo informa se aquele aplicativo pode ou não utilizar um login de um outro usuário já em **produção** (se aquele aplicativo pode requisitar dados de clientes que não sejam o dono do APP ou um desenvolvedor).

Para enviar seu aplicativo para produção, basta utilizar a opção **"Enviar para produção"** nas ações do seu aplicativo em API > Meus Aplicativos conforme abaixo:

![Opção de enviar aplicativo para produção](/docs/images/credentials/app-status.png)

**Atenção!** Para enviar um aplicativo para produção primeiro é necessário cadastrar uma URL válida de retorno para o login, para validarmos se está tudo pronto para usuários começarem a utilizar o seu APP:

![Enviar aplicativo para produção faltando URL de retorno](/docs/images/credentials/app-status-missing-returnurl.png)
